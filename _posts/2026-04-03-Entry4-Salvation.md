# Dear Reader,

I am sorry to inform you that Chirpy, the blogs old theme, is no more. I killed him. I don't regret killing Chirpy. He was good, but insufficient for my cause.

Please welcome minima! This is the default theme for a fresh jekyll installation. With a little bit of my own tweaking it's now a barren wasteland full of white, but we'll get there eventually.
I'm sure by the time you read this I have picked my own personal (usually awful) color scheme.

### Understanding

Today's theme is truly "understanding", not only did I figure out how to do a proper installation of jekyll, I also finally *get* how the file structure works.

I used to constantly edit the "site" directory and seeing all my changes turn to null and void whenever I rebuilt the site. Now I understand that the site directory is where files are written to. But where are they written from?

The "complicated" part was that the default themes are located in some far ass directory within the gems installation. The point is that you have to copy those default directories into your own repo. If those directories don't exist within the repo, jekyll will just default to the default ones.

This is a confusing process at first. I've never had to manually copy a directory into another directory to make changes like this. Why don't you just put them in the directory in the first place..?

> Fair point, me. Good job. Clarity to me is most important.
> The fact that these themes are completely uneditable and hard to find unless you know what you are looking for is a bit annoying to me.
> Unless it's the inner workings of how the software works, customizable features should not be hiding in installation directories.
>
> And to the creator of chirp, thanks a lot man, your theme is awesome (genuinely).
> I unfortunately did not end up using it because it was too complex to change things. Perhaps a simple config to change stuff would go a long way for people too lazy to learn css!

### Victory

To me this is victory. I understand now where my layouts are located and how to edit them. I even managed on my own to get this to work on my GitHub io. This is incredible to me because I feel like I *understand* what is happening. I no longer need a tutorial.

Now that I understand where things are, I can also actually start using HTML and CSS. Until this point all I could do was aimlessly browse through files with overly complex CSS code. Now I can make it myself, and I know where to find it.

I suppose the next step is to play around with HTML and CSS (Teach said I could take 1 extra day, which is great because I'm well into overtime right now).

Cheers to everyone writing tutorials and documentations. I may be overly critical and cynical about them, but I always appreciate the effort :]
